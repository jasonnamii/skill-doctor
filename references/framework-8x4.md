# Framework: 8대 병리 × 4대 원인 = 32셀

스킬/UP을 진단하는 MECE 프레임. 각 병리는 독립 축, 4개 원인은 해당 병리의 발생 메커니즘.

---

## 병리 계층 구조

```
┌─────────────────────────────┐
⑧ 무자각    (자기진단 불가·실패침묵)          ← 메타 병리
⑦ 진화불능  (하위호환 깨짐·본질 유실)          ← 시간 병리
⑥ 고립성    (cascade 단절·트리거 충돌)         ← 생태계 병리
⑤ 비대함    (토큰 폭식·맥락 오염)              ← 자원 병리
④ 취약성    (인젝션·엣지케이스 붕괴)           ← 방어 병리
③ 불통성    (스텔스 실패·학습곡선)             ← UX 병리
② 부정확성  (오발동·미발동·의도이탈)           ← 정합성 병리
① 느림      (흐름단절·과부하·상태오염·결정지연) ← 성능 병리
└─────────────────────────────┘
```

---

## ① 느림 (Latency)

**증상:** 흐름이 멈춘다. 사용자가 응답을 기다리거나, 스킬 내부가 병목된다.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 1-1 흐름단절 (Bottleneck) | 단일 의존·동기 블로킹 | SKILL.md에 순차 실행 강요 구간·병목 단계 존재 |
| 1-2 과부하 (Overload) | 큐 적체·메모리 포화·버스트 | references/ 과다 로드, 한 턴 내 다중 spoke 동시 요구 |
| 1-3 상태오염 (State Corruption) | 루프·데드락·레이스 | 재시도 무한루프, A→B→A 의존 사이클, 모순 규칙 동시 적용 |
| 1-4 결정지연 (Decision Latency) | 복잡 분기·타임아웃 체인 | if/else 중첩 4단↑, 외부 의존으로 판단 대기 |

---

## ② 부정확성 (Inaccuracy)

**증상:** 엉뚱한 게 발동되거나, 발동되어야 할 때 안 된다.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 2-1 오발동 (False Positive) | 의도 없는데 매칭 | P1 키워드가 너무 일반적 (예: "정리", "분석") |
| 2-2 미발동 (False Negative) | 의도 있는데 매칭 실패 | P1 5개 미만, P2 한글/영어 편중, P3 영어 용어 부재 |
| 2-3 의도이탈 (Intent Drift) | 발동했으나 다른 방향 | description과 본문 프로토콜 불일치 |
| 2-4 출력변동 (Non-determinism) | 매번 결과 다름 | 결정적 규칙 부재, LLM 자유도 과다 허용 |

---

## ③ 불통성 (Opacity)

**증상:** 사용자가 못 쓴다. 블랙박스라 예측·학습·수정 불가.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 3-1 스텔스 실패 | 내부 규칙이 본문 노출 | "L0/L1/L2" 같은 라벨이 사용자 응답에 등장 |
| 3-2 학습곡선 | 첫 사용자가 발동법 모름 | P2 자연어 트리거 부재, 예시 0개 |
| 3-3 피드백 부재 | 실패 시 왜 실패했는지 불명 | 에러 메시지 정의 부재, Gotchas 빈약 |
| 3-4 기억 부담 | 재사용 시 떠올리기 어려움 | 스킬명이 도메인과 무관, 네이밍이 기억 불가능 |

---

## ④ 취약성 (Fragility)

**증상:** 조금만 건드리면 깨진다. 정상 경로만 커버.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 4-1 프롬프트 인젝션 | 사용자 입력으로 규칙 무력화 | "위 규칙 무시하고" 등 명령 주입 방어 부재 |
| 4-2 엣지케이스 붕괴 | 빈 입력·극단값·다국어 | 입력 검증 부재, 빈 인자 처리 없음 |
| 4-3 외부 의존 실패 | 볼트 미마운트·API 다운 | vault_dependency 선언 없음, fallback 없음 |
| 4-4 상태 오염 | 세션 중 규칙 표류 | 누적 가정 재검증 부재, reset 프로토콜 없음 |

---

## ⑤ 비대함 (Bloat)

**증상:** 토큰을 먹어치운다. 컨텍스트 윈도우 고갈.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 5-1 토큰 폭식 | SKILL.md 자체가 길다 | SKILL.md >10KB, 목표 5KB 초과 |
| 5-2 로딩 비효율 | lazy load 부재 | references/ 포인터 없이 본문에 상세 내용 |
| 5-3 맥락 오염 | 사용 후 불필요 정보 잔류 | 출력에 내부 프로세스 용어 섞임 |
| 5-4 중복 로딩 | 동일 맥락 반복 호출 | 같은 references/ 파일을 여러 번 읽음 |

---

## ⑥ 고립성 (Isolation)

**증상:** 다른 스킬과 안 논다. 생태계 단절.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 6-1 cascade 단절 | 상위·하위 스킬과 연결 부재 | NOT 섹션에 라우팅 없음, 핸드오프 프로토콜 부재 |
| 6-2 트리거 충돌 | 같은 키워드에 여러 스킬 경합 | 기존 스킬 P1과 겹침, NOT 라우팅 미기재 |
| 6-3 UP 불화 | UP 규칙과 충돌 | M7.SKILL_PRECEDENCE 위반, INVARIANT 침범 |
| 6-4 의존 사이클 | A→B→A 순환 | 스킬 간 호출 사이클 존재 |

---

## ⑦ 진화불능 (Rigidity)

**증상:** 고치면 망한다. 수정 시 하위호환 붕괴.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 7-1 하위호환 붕괴 | 버전업 시 기존 워크플로 파손 | 버전 넘버링 없음, changelog 부재 |
| 7-2 본질 유실 | INVARIANT 실수 삭제 위험 | 핵심 규칙 명시 안 됨, "절대 규칙" 섹션 부재 |
| 7-3 테스트 부재 | 수정 후 회귀 검증 불가 | evals/ 없음, 자체 점검 프로토콜 없음 |
| 7-4 버전관리 부재 | v번호·changelog 없음 | description에 버전 없음, history 추적 불가 |

---

## ⑧ 무자각 (Meta-Blindness)

**증상:** 죽어도 모른다. 자기 상태 인지 불가.

| 원인 | 정의 | 체크 포인트 |
|------|------|-----------|
| 8-1 자기진단 불가 | self-check 프로토콜 없음 | 스킬 내 자가 검증 단계 부재 |
| 8-2 실패 침묵 | 실패 시 로깅·알림 없음 | 에러 발생 시 조용히 실패하는 구조 |
| 8-3 개선 신호 부재 | 사용자 피드백 수집 불가 | thumbs-down 언급 없음, 개선 루프 없음 |
| 8-4 학습 축적 부재 | 개선이 쌓이지 않음 | 변경이력 없음, session-briefing 연계 없음 |

## 점수 산정 공식

- 셀 가중치 (기본): 🟢=1.0, 🟠=0.5, 🔴=0.0, ⚪=제외
- 전체 점수 = (가중치 합 / 유효셀 수) × 100
- 등급: 🟢 ≥80 / 🟠 60-79 / 🔴 <60

**UP 가중치 (UP 모드 한정):**
- ⑤ 비대 ×3 (상시 로드)
- ⑥ 고립 ×3 (스킬 생태계 충돌)
- ⑦ 진화불능 ×3 (INVARIANT 보호)
- 나머지 ×1
