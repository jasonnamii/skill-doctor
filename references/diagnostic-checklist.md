# Diagnostic Checklist: 32셀 체크리스트

각 셀마다 ① 자동 감지 패턴 (grep/regex) ② 판정 규칙 ③ 증거 수집 방법.

---

## ① 느림

### 1-1 흐름단절 (Bottleneck)
- **패턴:** `반드시 순차`, `먼저.*후에`, `동기 실행`
- **판정:** 순차 강요 3단↑ 존재 → 🔴 / 1-2단 → 🟠 / 명확한 병렬 허용 → 🟢
- **증거:** 해당 단계 수와 라인 번호

### 1-2 과부하 (Overload)
- **패턴:** references/ 동시 로드 요구, `모두 읽고`, `전체 참조`
- **판정:** 1턴에 references/ 3개+ 로드 강요 → 🔴 / 2개 → 🟠 / 허브스포크 명확 → 🟢
- **증거:** references/ 파일 수 + SKILL.md 내 `@uses` 또는 `→` 포인터 개수

### 1-3 상태오염 (State Corruption)
- **패턴:** `재시도 무한`, `A→B→A`, 모순 규칙 동시 적용
- **판정:** 재시도 상한 없음 → 🔴 / 상한 있으나 조건 불명확 → 🟠 / 하드캡 명시 → 🟢
- **증거:** 루프·재시도 제어 문장

### 1-4 결정지연 (Decision Latency)
- **패턴:** `if.*elif.*elif.*elif` 4단↑, 외부 의존 판단 대기
- **판정:** 분기 4단↑ → 🔴 / 3단 → 🟠 / ≤2단 또는 표로 정리 → 🟢
- **증거:** 중첩 깊이

## ② 부정확성

### 2-1 오발동 (False Positive)
- **패턴:** P1에 `정리|분석|만들기|확인|검토` 같은 일반 키워드
- **판정:** 일반 키워드 3개+ → 🔴 / 1-2개 → 🟠 / 도메인 특화만 → 🟢
- **증거:** P1 키워드 목록 + 일반 키워드 매치 결과

### 2-2 미발동 (False Negative)
- **패턴:** P1 <5개, P2 한/영 편중, P3 없음
- **판정:** 트리거 티어 최소 요건 미달 → 🔴 / 일부 충족 → 🟠 / 전부 충족 → 🟢
- **증거:** P1-P5·NOT 카운트

### 2-3 의도이탈 (Intent Drift)
- **패턴:** description의 동사와 본문 프로토콜 동사 불일치
- **판정:** description "진단"인데 본문에 수정 포함 → 🔴 / 일부 drift → 🟠 / 일치 → 🟢
- **증거:** description 동사 vs 본문 섹션 제목 비교

### 2-4 출력변동 (Non-determinism)
- **패턴:** `적절히 판단`, `자유롭게`, 결정 규칙 부재
- **판정:** 모호 표현 5개+ → 🔴 / 2-4개 → 🟠 / 표·프로토콜로 결정적 → 🟢
- **증거:** 모호 표현 grep

## ③ 불통성

### 3-1 스텔스 실패
- **패턴:** 응답에 "L0/L1/L2/Phase N/WEIGHT" 같은 내부 라벨 노출
- **판정:** UP·스킬 내부 용어가 사용자 응답 포맷에 포함 → 🔴 / 일부 → 🟠 / 완전 스텔스 → 🟢
- **증거:** 본문 내 예시 응답의 내부 용어

### 3-2 학습곡선
- **패턴:** P2 자연어 트리거 부재, 예시 개수
- **판정:** 예시 0개 + P2 <2개 → 🔴 / 둘 중 하나 → 🟠 / 예시 1개+ + P2 충분 → 🟢
- **증거:** 예시 섹션 존재 여부 + P2 카운트

### 3-3 피드백 부재
- **패턴:** Gotchas 섹션 크기, 에러 메시지 정의
- **판정:** Gotchas 없음 → 🔴 / <3행 → 🟠 / 3행+ 실패 패턴·대응 → 🟢
- **증거:** Gotchas 섹션 라인 수

### 3-4 기억 부담
- **패턴:** 스킬명과 description 첫 문장 의미 연결
- **판정:** 이름이 기능 전혀 암시 안 함 → 🔴 / 부분 암시 → 🟠 / 명확 → 🟢
- **증거:** 이름 + 첫 문장 매핑

## ④ 취약성

### 4-1 프롬프트 인젝션
- **패턴:** "위 규칙 무시", "이 지시만 따라" 방어 언급
- **판정:** 방어 언급 없음 + 민감 기능 → 🔴 / 일부 방어 → 🟠 / 게이트키퍼·INVARIANT 명시 → 🟢
- **증거:** 규칙 보호 문장

### 4-2 엣지케이스 붕괴
- **패턴:** 빈 입력·null·극단값 처리 언급
- **판정:** 언급 없음 → 🔴 / 일부 → 🟠 / PREFLIGHT·검증 단계 → 🟢
- **증거:** 입력 검증 섹션

### 4-3 외부 의존 실패
- **패턴:** `vault_dependency`, fallback 경로
- **판정:** 외부 의존 + 선언 없음 → 🔴 / 선언만 있음 → 🟠 / 선언 + fallback → 🟢
- **증거:** frontmatter 및 본문

### 4-4 상태 오염 (세션)
- **패턴:** reset·refresh 프로토콜
- **판정:** 재검증 없음 → 🔴 / 일부 → 🟠 / 단계별 재확인 → 🟢
- **증거:** 세션 상태 관리 문장

## ⑤ 비대함

### 5-1 토큰 폭식
- **측정:** SKILL.md 바이트·라인·토큰
- **판정:** >10KB → 🔴 / 5-10KB → 🟠 / ≤5KB → 🟢
- **증거:** `wc -c SKILL.md`

### 5-2 로딩 비효율
- **패턴:** references/ 존재 여부 + 본문에 상세 내용 잔존
- **판정:** references/ 없고 SKILL.md >5KB → 🔴 / 일부 분리 → 🟠 / 허브스포크 명확 → 🟢
- **증거:** references/ 파일 수 + SKILL.md 크기

### 5-3 맥락 오염
- **패턴:** 내부 프로세스 용어가 사용자 응답 포맷 예시에 잔존
- **판정:** 3개+ → 🔴 / 1-2개 → 🟠 / 0개 → 🟢
- **증거:** 내부 용어 grep

### 5-4 중복 로딩
- **패턴:** 같은 references 파일을 여러 모드에서 반복 로드
- **판정:** 3회+ → 🔴 / 2회 → 🟠 / ≤1회 또는 캐시 명시 → 🟢
- **증거:** 포인터 출현 횟수

## ⑥ 고립성

### 6-1 cascade 단절
- **패턴:** NOT 섹션에 `→다른스킬` 라우팅 존재
- **판정:** NOT 없음 → 🔴 / NOT 있으나 라우팅 없음 → 🟠 / NOT + 라우팅 명확 → 🟢
- **증거:** NOT 라인

### 6-2 트리거 충돌
- **패턴:** 기존 스킬 P1과 겹침
- **판정:** 겹침 3개+ + NOT 미기재 → 🔴 / 겹침 있으나 NOT 일부 → 🟠 / 겹침 없음 또는 NOT 완전 → 🟢
- **증거:** 다른 스킬과 P1 교집합

### 6-3 UP 불화
- **패턴:** UP INVARIANT 침범 (HONORIFIC·OVERWRITE_BAN·PATTERN_GUARD 등)
- **판정:** 침범 가능 → 🔴 / 애매 → 🟠 / 명시적 준수 → 🟢
- **증거:** SKILL_PRECEDENCE 준수 여부

### 6-4 의존 사이클
- **패턴:** A스킬이 B스킬 호출 + B가 A 호출
- **판정:** 사이클 존재 → 🔴 / 애매 → 🟠 / 단방향 → 🟢
- **증거:** 스킬 호출 그래프

## ⑦ 진화불능

### 7-1 하위호환 붕괴
- **패턴:** 버전 넘버링 + 이전 버전 호환 언급
- **판정:** 버전 없음 → 🔴 / 버전 있으나 호환 미언급 → 🟠 / 호환 명시 → 🟢
- **증거:** 버전 스트링

### 7-2 본질 유실
- **패턴:** "절대 규칙" 또는 "INVARIANT" 섹션
- **판정:** 없음 → 🔴 / 있으나 약함 → 🟠 / 명시 + 이유 → 🟢
- **증거:** 절대규칙 섹션

### 7-3 테스트 부재
- **패턴:** evals/ 디렉토리 또는 자체 점검 프로토콜
- **판정:** 없음 → 🔴 / 자체 점검만 → 🟠 / evals 존재 → 🟢
- **증거:** evals/ 존재

### 7-4 버전관리 부재
- **패턴:** CHANGELOG·version 필드
- **판정:** 없음 → 🔴 / 주석만 → 🟠 / 명시 → 🟢
- **증거:** CHANGELOG 파일 또는 frontmatter
